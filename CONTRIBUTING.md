# Вклад в развитие проекта

Спасибо за интерес к проекту! Мы приветствуем вклады от сообщества и ценим любую помощь в развитии проекта.

## 📋 Как можно помочь

- 🐛 **Сообщить об ошибке** — создайте issue с подробным описанием проблемы
- 💡 **Предложить улучшение** — опишите идею в issue перед началом работы
- 🔧 **Исправить баг** — выберите open issue и отправьте PR
- 📝 **Улучшить документацию** — исправления опечаток, примеры использования
- 🧪 **Добавить тесты** — особенно для security-сценариев
- 🔍 **Аудит безопасности** — поиск уязвимостей

## 🚀 Процесс внесения изменений

### 1. Форк и клонирование
```bash
git clone https://github.com/YOUR_USERNAME/system_info.git
cd system_info
```

### 2. Создание ветки
```bash
git checkout -b feature/your-feature-name
# или
git checkout -b fix/issue-number
```

### 3. Внесение изменений
- Следуйте принципам чистого кода
- Добавляйте тесты для нового функционала
- Обновляйте документацию при необходимости

### 4. Запуск тестов
```bash
python -m pytest test_system_info.py -v
```

### 5. Commit сообщения
- Используйте понятные описания
- Ссылайтесь на issue: `Fix #123: add input validation`
- Следуйте формату: `type: description` (feat, fix, docs, test, refactor)

### 6. Pull Request
- Создайте PR в основную ветку
- Опишите изменения и ссылку на issue
- Дождитесь code review

## 📏 Стандарты кода

- **Безопасность прежде всего** — все входные данные должны валидироваться
- **PEP 8** — следуйте стилю кодирования Python
- **Типизация** — используйте type hints где возможно
- **Документирование** — docstrings для публичных функций
- **Тестирование** — минимальное покрытие 80%

## 🔒 Требования безопасности

При внесении изменений обязательно:
- ✅ Валидируйте все входные данные
- ✅ Обрабатывайте конкретные типы исключений
- ✅ Не используйте опасные функции без проверки
- ✅ Добавляйте тесты на security-сценарии
- ✅ Проверяйте пути на path traversal

## 💬 Общение

- Используйте вежливый и инклюзивный тон
- Следуйте [Кодексу поведения](CODE_OF_CONDUCT.md)
- Задавайте вопросы в issue или обсуждениях
